⚡ Module 07 · Lesson 02: Index Optimisation

Choose and validate index strategies for read-heavy and mixed workloads

Advanced

📖 Lesson Overview

Indexes can significantly reduce query time when they match filter and join patterns.

🔑 Key Concepts

  • Single-column, composite, unique, and partial indexes serve different workloads.
  • Indexing every column is counterproductive for writes and storage.

💻 SQL Examples

Types of Indexes

-- Single column
CREATE INDEX idx_orders_machine ON production_orders(machine_id);

-- Composite (multi-column) - order matters!
CREATE INDEX idx_orders_machine_time ON production_orders(machine_id, start_time);

-- Unique
CREATE UNIQUE INDEX idx_parts_code ON spare_parts(part_code);

-- Partial index - only index active machines
CREATE INDEX idx_active_machines ON machines(machine_id)
WHERE status = 'Active';

Check Existing Indexes

SELECT indexname, indexdef
FROM pg_indexes
WHERE tablename = 'production_orders';

🛡️ Notes / Best Practices

✅ Index columns used in WHERE, JOIN ON, ORDER BY, and GROUP BY.

❌ Avoid indexing small tables, low-cardinality columns, or rarely queried fields.
