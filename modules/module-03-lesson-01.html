📝 Module 03 · Lesson 01: Writing & Modifying Data (DML)

Work with INSERT, UPDATE, and DELETE safely in production-style tables

Beginner

📖 Lesson Overview

This lesson introduces core DML operations used to add, modify, and remove rows.

🔑 Key Concepts

  • INSERT: add one or many rows in a single statement.
  • UPDATE: modify existing rows with precise filters.
  • DELETE: remove targeted rows while preserving needed data.
  • Always validate impacted rows before destructive changes.

💻 SQL Examples

INSERT

-- Single row insert
INSERT INTO machines (machine_name, location, status, install_date)
VALUES ('Laser Cutter D', 'Workshop 2', 'Active', '2024-01-10');

-- Multiple rows
INSERT INTO machines (machine_name, location, status, install_date)
VALUES
  ('Press E', 'Line 3', 'Active', '2023-06-01'),
  ('Welder F', 'Line 1', 'Inactive', '2021-03-15');

UPDATE

-- Always use WHERE or you update every row!
UPDATE machines
SET status = 'Inactive'
WHERE machine_id = 3;

-- Update multiple columns
UPDATE machines
SET status = 'Active', last_maintenance = NOW()
WHERE machine_name = 'Conveyor C';

DELETE

-- Delete a specific row
DELETE FROM downtime_logs
WHERE log_id = 42;

-- Delete all logs older than 1 year
DELETE FROM downtime_logs
WHERE log_date < NOW() - INTERVAL '1 year';

🛡️ Notes / Best Practices

⚠️ Always run a SELECT with the same WHERE clause first to verify what will be affected before running UPDATE or DELETE.
