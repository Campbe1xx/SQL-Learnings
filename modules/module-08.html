📚 Module 08 – Security & Access Control

Users, roles, privileges, Row-Level Security (RLS), and SQL injection prevention

Advanced

📖 Lesson Overview

In this module, you'll learn:

  • Users & Roles – Create and manage database accounts
  • Privileges – GRANT/REVOKE permissions on objects
  • Row-Level Security (RLS) – Restrict rows per user/role
  • SQL Injection Prevention – Parameterized queries and prepared statements
  • Audit & Logging – Track changes and access
  • Password & Connection Security – Best practices

📋 Solution 1

Q1 Solution:

-- Create read-only role CREATE ROLE read_only_user WITH NOLOGIN; -- Grant permissions GRANT USAGE ON SCHEMA public TO read_only_user; GRANT SELECT ON employee_data TO read_only_user; -- To assign this role to a user: -- GRANT read_only_user TO actual_user_name;

Q2 Solution:

-- Create analyst role CREATE ROLE analyst_role WITH NOLOGIN; -- Grant permissions GRANT USAGE ON SCHEMA public TO analyst_role; GRANT SELECT, INSERT, UPDATE ON employee_data TO analyst_role; -- Revoke UPDATE REVOKE UPDATE ON employee_data FROM analyst_role; -- Now analyst_role has only SELECT and INSERT

Q3 Solution:

-- Enable RLS on the table ALTER TABLE employee_data ENABLE ROW LEVEL SECURITY; -- Create a policy allowing only Engineering department access CREATE POLICY engineering_policy ON employee_data USING (department = 'Engineering') WITH CHECK (department = 'Engineering'); -- Create a default DENY policy for other roles (optional but recommended) CREATE POLICY deny_all ON employee_data AS PERMISSIVE FOR SELECT USING (FALSE); -- Assign policy to specific role ALTER TABLE employee_data FORCE ROW LEVEL SECURITY;

C1 Solution:

-- VULNERABLE: SQL Injection Risk -- DO NOT USE THIS in production: -- CREATE FUNCTION get_employee_unsafe(p_dept_name VARCHAR) -- RETURNS TABLE(...) AS $$ -- BEGIN -- RETURN QUERY EXECUTE 'SELECT * FROM employee_data WHERE department = ''' || p_dept_name || '''; -- END; -- $$ LANGUAGE plpgsql; -- -- An attacker could call: -- SELECT * FROM get_employee_unsafe('Engineering''; DROP TABLE employee_data; --') -- -- SAFE: Use parameterized queries CREATE OR REPLACE FUNCTION get_employee_safe(p_dept_name VARCHAR) RETURNS TABLE (emp_id INT, emp_name VARCHAR, department VARCHAR, salary NUMERIC, created_at TIMESTAMP) LANGUAGE plpgsql AS $$ BEGIN RETURN QUERY SELECT e.emp_id, e.emp_name, e.department, e.salary, e.created_at FROM employee_data e WHERE e.department = p_dept_name; -- Parameter binding prevents injection END; $$; -- Safe call: SELECT * FROM get_employee_safe('Engineering');
